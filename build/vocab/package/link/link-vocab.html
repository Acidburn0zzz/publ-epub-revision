1. Overview

1.1 Purpose and Scope

This document defines relationship values and properties for use with the EPUBÂ® Package Document link element.

1.2 Conformance Statements

The keywords MUST, MUST NOT, REQUIRED, SHALL, SHALL NOT, SHOULD, SHOULD NOT, RECOMMENDED, MAY, and OPTIONAL in this document are to be interpreted as described in [RFC2119].

All examples are informative.

2. Link Relationships

The following values can be used with the Package Document link element to establish the relationship between the resource referenced in the element's href attribute and the EPUB Publication.

2.1 acquire

Name: acquire
Description: The acquire relationship is used with EPUB Previews to identify where the full version of the Publication can be acquired.
Cardinality: Zero or more
Extends: Only applies to the Publication. Must not be used when the refines attribute is present.
Example: <link rel="acquire" href="http://example.org/book/9781448103706" media-type="text/html"/>

2.2 marc21xml-record [DEPRECATED]

Name: marc21xml-record
Use of the marc21xml-record relationship is deprecated. It is replaced by the record relationship with the media-type value "application/marc".

2.3 mods-record [DEPRECATED]

Name: mods-record
Use of the mods-record relationship is deprecated. It is replaced by the record relationship with the properties attribute value mods.

2.4 onix-record [DEPRECATED]

Name: onix-record
Use of the onix-record relationship is deprecated. It is replaced by the record relationship with the properties attribute value onix.

2.5 record

Name: record
Description:

Indicates that the referenced resource is a metadata record.

The type of record referenced must be identifiable from the link element's media-type attribute.
Cardinality: Zero or more
Extends: Only applies to the Publication. Must not be used when the refines attribute is present.
Example: <link rel="record" href="book/52.atom" media-type="application/atom+xml;type=entry;profile=opds-catalog"/>

2.6 xml-signature

Name: xml-signature
Description:

The xml-signature relationship indicates the referenced resource contains an XML Signature [XML DSIG Core] for the Publication or a resource associated with the property.

The link element's href attribute typically references a specific signature element in the Signatures.xml file [OCF3] via a fragment identifier.
Cardinality: Zero or more
Extends: All properties.
Example: <link refines="#meta-authority-01" rel="xml-signature" href="../META-INF/signatures.xml#MAI-Signature"/>

2.7 xmp-record [DEPRECATED]

Name: xmp-record
Use of the xmp-record relationship is deprecated. It is replaced by the record relationship with the properties attribute value xmp.

3. Link Properties

The following values can be used with the Package Document link element to establish the type of record a referenced resource represents. These values are provided for record formats that cannot be uniquely identified by their media type.

3.1 mods [DRAFT]

Name: mods
Description: The mods property indicates the referenced resource is a MODS record [MODS].
Cardinality: Zero or one
Extends: Only applies to the Publication. Must not be used when the refines attribute is present.
Example: <link rel="record" href="pub/meta/nor-wood-mods.xml"/ properties="mods">

3.2 onix [DRAFT]

Name: onix
Description: The onix property indicates the referenced resource is an ONIX record [ONIX].
Cardinality: Zero or one
Extends: Only applies to the Publication. Must not be used when the refines attribute is present.
Example: <link rel="record" href="pub/meta/nor-wood-onix.xml" properties="onix"/>

3.3 xmp [DRAFT]

Name: xmp
Description: The xmp property indicates the referenced resource is an XMP record [XMP].
Cardinality: Zero or one
Extends: Only applies to the Publication. Must not be used when the refines attribute is present.
Example: <link rel="record" href="pub/meta/nor-wood-xmp.xml properties="xmp"/>
